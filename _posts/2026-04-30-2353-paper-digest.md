# 从闭环训练到可审计数据智能

> **TL;DR：** 本期我重点看 5 篇 2026 年 4 月底的新稿。共同线索不是“更会聊天的模型”，而是让模型进入可验证的闭环：在本地工作区里训练 agent，在真实未来事件里拿延迟奖励，用世界模型生成空间转移监督，用过程奖励模型审计数据分析步骤，再用多阶段指标审计企业文档 RAG。最近 8 小时内没有检索到足够匹配的新论文，所以本期扩展到最近 3-7 天，并优先收录 2026-04-29 的论文。

## 本期我在看什么

这几篇论文让我感觉 agent 研究正在从“把 LLM 接上工具”转向更工程化的三个问题：

1. 训练信号从哪里来：是合成工作区任务、真实世界延迟结果、世界模型生成的未来视角，还是可执行环境中的过程级审计？
2. 评价是否真的能约束行为：只看最终答案不够，还要看中间步骤、工作区状态、检索和生成链条、以及 agent 是否能从失败中恢复。
3. 小模型是否能靠任务特定数据补足规模差距：ClawGym、DataPRM 和 World2VLM 都在不同场景里给了正面证据，但也都把验证器、教师模型或数据分布的脆弱性暴露出来。

初筛时我还看到了 Bian Que、GLM-5V-Turbo、AgentSim、STARRY、X-WAM、FCMBench-Video、DataClaw 等候选。最后选择下面 5 篇，是因为它们分别覆盖 agentic training、world models、data agents、document intelligence，并且都有可访问的 arXiv HTML 全文可细读。

## 1. ClawGym：把个人工作区 agent 的训练、验证和 benchmark 连成一条线

**论文：** [ClawGym: A Scalable Framework for Building Effective Claw Agents](https://arxiv.org/abs/2604.26904)  
**作者：** Fei Bai, Huatong Song, Shuang Sun, Daixuan Cheng, Yike Yang, Chuan Hao, Renyuan Li, Feng Chang, Yuan Wei, Ran Tao, Bryan Dai, Jian Yang, Wayne Xin Zhao  
**机构：** 中国人民大学高瓴人工智能学院、IQuest Research、北京航空航天大学  
**日期：** arXiv, 2026-04-29  
**全文：** [arXiv HTML](https://arxiv.org/html/2604.26904v1)

**核心图表：**

![ClawGym-SynData pipeline](https://arxiv.org/html/2604.26904v1/x1.png)

*图 1：ClawGym-SynData 的任务合成、资源准备、混合验证和质量过滤流水线。*

![ClawGym RL curves](https://arxiv.org/html/2604.26904v1/x4.png)

*图 3：基于 sandbox-parallel rollouts 的 RL 训练曲线。*

**图表线索：** 表 6 比较 ClawGym-Agents、Qwen 系列、开源 frontier 模型和闭源 frontier 模型在 ClawGym-Bench 与 PinchBench 上的表现；表 7 分析 persona-driven 与 skill-grounded 两种任务合成策略的互补性。

**我读到的重点：** ClawGym 针对的是“Claw-style”个人工作区 agent：模型要在本地文件、工具、持久状态和多步任务之间来回操作。论文的关键不是单个 benchmark，而是三件事连在一起：ClawGym-SynData 合成 13.5K 个过滤后的工作区任务；ClawGym-Agents 用黑盒 rollout 轨迹做 SFT，并进一步尝试 GRPO；ClawGym-Bench 用 200 个经过自动过滤和人机复核的任务做评测。

方法上，这篇把任务来源拆成两路：persona-driven 生成真实用户意图，skill-grounded 约束任务必须落在 agent 的实际操作边界里。验证也不是纯文本评分，而是 code-based verifier 与 rubric-based judge 的混合。这个设计对我比较重要，因为它避免了 agent 论文里常见的一个问题：任务听起来真实，但最后只能靠模型自评。

实验上，ClawGym-4B、8B、30A3B 在 ClawGym-Bench 上都超过各自 Qwen baseline。论文还报告 ClawGym-30A3B 在 PinchBench 上达到 86.00，并且在 ClawGym-Bench 平均分超过更大的 Qwen3-235A23B。RL 部分使用 2,000 个合成任务、每个 prompt 8 个 rollout、100 步训练，展示了 vanilla 4B 和 SFT 后模型都能继续受益。

**我会优先看这篇的原因：** 它把“agentic training”落到了可执行工作区，而不是停留在对话轨迹。对个人科研助理、数据清洗助理、网站维护助理这类场景，真正难的是文件状态、工具错误恢复、长上下文依赖和验证器设计。

**局限和问题：** 资源在论文里标注为即将发布，复现实验还要等公开版本。合成任务质量依赖 verifier 与 reviewer 的设计，真实用户工作区的长尾状态也可能比 benchmark 更乱。下一步我会关注：ClawGym-Bench 是否能被外部 agent 框架稳定复现，rubric-based judgment 与 code verifier 冲突时如何裁决，以及 RL 是否会学会“迎合 verifier”。

**主题连接：** agentic training、可验证工作区任务、sandbox rollout、个人 agent。

## 2. FutureWorld：用真实未来事件给预测 agent 提供延迟奖励

**论文：** [FutureWorld: A Live Environment for Training Predictive Agents with Real-World Outcome Rewards](https://arxiv.org/abs/2604.26733)  
**作者：** Zhixin Han, Yanzhi Zhang, Chuyang Wei, Maohang Gao, Xiawei Yue, Kefei Chen, Yu Zhuang, Haoxiang Guan, Jiyan He, Jian Li, Yitong Duan, Yu Shi, Mengting Hu, Shuxin Zheng  
**机构：** 南开大学软件学院、中国科学院数学与系统科学研究院、北京中关村学院、清华大学  
**日期：** arXiv, 2026-04-29  
**全文：** [arXiv HTML](https://arxiv.org/html/2604.26733v1)；[数据集页面](https://huggingface.co/datasets/PredictingFuture/FutureWorld)

**核心图表：**

![FutureWorld question pipeline](https://arxiv.org/html/2604.26733v1/x2.png)

*图 2：从公开来源收集未来事件、构造问题、过滤和重采样的流程。*

![FutureWorld training loop](https://arxiv.org/html/2604.26733v1/x3.png)

*图 3：agent 预测、轨迹记录、真实结果回收和奖励回放的训练闭环。*

![FutureWorld checkpoint performance](https://arxiv.org/html/2604.26733v1/x4.png)

*图 4：连续训练多天后，不同 checkpoint 在同一天预测集上的表现。*

**我读到的重点：** FutureWorld 把“未来预测”建成了 live environment。系统每天 20:00 发布一批第二天有望揭晓的问题；agent 通过搜索和推理给出概率预测；等真实结果可获取后，环境把 outcome 写回轨迹，并用负 Brier loss 做 reward。论文里每个问题采样 4 条随机轨迹，训练用 GRPO。

这篇最有价值的地方是它把“没有答案泄漏”的实时任务变成了训练信号。很多预测 benchmark 一旦冻结，就会被训练语料污染；FutureWorld 的实时问题天然更难被提前记忆。它同时承认了现实世界反馈的麻烦：连续 5 天观察中，预定时间平均有 35.65% 的问题拿不到 ground truth；晚一天再取，未解决比例降到 20%。这些未解决问题被丢弃，简单可行，但浪费监督信号。

实验上，论文训练 Qwen3-4B-Instruct、Qwen2.5-3B-Instruct 和 DeepSeek-R1-0528-Qwen3-8B 等 agent，并在 8 天训练后比较每日 checkpoint。图 4 显示 accuracy、Brier score 和 ECE 都随训练改善；图 6 进一步显示每日问题数量增大时，Qwen2.5-3B-Instruct 有正向 scaling trend。

**我会优先看这篇的原因：** 它把 agent 训练从静态题库推进到“真实世界延迟反馈”。如果未来要做科研预测、市场事件预测或实验规划 agent，问题不只是模型会不会搜索，而是它能不能在不确定结果揭晓后吸收校准信号。

**局限和问题：** 真实事件的答案发布时间不稳定，丢弃未解决样本可能引入选择偏差；搜索工具观测被作为上下文但不直接拿 policy loss，credit assignment 仍然很粗；benchmark 还会受周末、假期和来源更新周期影响。后续我会关注它是否引入 longer-delay replay buffer、问题难度控制，以及数值预测与多选预测的奖励是否需要更细分。

**主题连接：** agentic training、真实世界 reward、online environment、概率校准。

## 3. World2VLM：把世界模型的“想象”蒸馏进 VLM，而不是推理时再调用它

**论文：** [World2VLM: Distilling World Model Imagination into VLMs for Dynamic Spatial Reasoning](https://arxiv.org/abs/2604.26934)  
**作者：** Wanyue Zhang, Wenxiang Wu, Wang Xu, Jiaxin Luo, Helu Zhi, Yibin Huang, Shuo Ren, Zitao Liu, Jiajun Zhang  
**机构：** 中国科学院自动化研究所、中国科学院大学人工智能学院、清华大学、哈尔滨工业大学、武汉人工智能研究院  
**日期：** arXiv, 2026-04-29  
**全文：** [arXiv HTML](https://arxiv.org/html/2604.26934v1)；[代码](https://github.com/WanyueZhang-ai/World2VLM)；[数据集](https://huggingface.co/datasets/WanyueZhang/World2VLM)

**核心图表：**

![World2VLM overview](https://arxiv.org/html/2604.26934v1/x2.png)

*图 2：用世界模型生成动作对齐的未来视角，再构造正向与反向空间推理任务。*

![World2VLM scaling curve](https://arxiv.org/html/2604.26934v1/x3.png)

*图 3：世界模型生成数据从 0 到 100K 时，多个动态空间推理 benchmark 的分数变化。*

**图表线索：** 表 2 比较 SVC 与 HY-WorldPlay 两类 teacher world model 下的主结果；表 3 拆解 SAT-Real 的 egocentric movement、object movement、goal aim、action consequence、perspective taking；表 4 做 bidirectional distillation 与训练来源消融。

**我读到的重点：** 这篇的核心判断很清楚：世界模型不一定只是在推理时被调用的外部 imagination module，也可以作为训练时 teacher。World2VLM 从一个初始视角和参数化相机轨迹出发，用世界模型生成几何对齐的未来视角，然后构造成两类监督：inverse task 让 VLM 从视角变化反推出动作；forward task 让 VLM 从动作预测空间后果。训练分两段：SFT 蒸馏，再用 task-aware GRPO 调整结构化输出。

实验用 Qwen2.5-VL-7B 作为 base，在 SAT-Real、SAT-Synthesized、VSI-Bench、MindCube 上评测。论文报告 World2VLM 在四个 benchmark 上都有提升，并且优于推理时耦合世界模型的 MindJourney-style baseline，同时部署时只需要普通 VLM 推理。消融里，forward-only 与 inverse-only 各有偏强场景，合起来平均最好；真实场景和模拟场景数据也互补。

我比较喜欢论文附录里对 teacher failure 的讨论。SVC 更擅长保持相机几何一致，但在大视角变化下会有局部渲染伪影；HY-WorldPlay 视觉上更自然，但可能把精确相机运动平滑或重解释。作者没有把其中一个 teacher 说成“全面更好”，而是承认不同 teacher 注入不同空间偏置。

**我会优先看这篇的原因：** 它把 world models 和 multimodal reasoning 的关系讲得更像训练数据工程问题：如何从可控生成器里提取动作条件转移，而不是每次推理都昂贵地生成未来画面。

**局限和问题：** teacher 质量仍然是瓶颈。当前过滤主要是规则式的，保留下来的样本基本被同等信任；如果 teacher 在几何、物体身份或可见性上出错，监督会直接变噪声。下一步值得看的是 reliability-weighted distillation、多 teacher agreement，以及按任务选择 teacher。

**主题连接：** world models、动态空间推理、VLM post-training、训练时 imagination distillation。

## 4. DataPRM：数据分析 agent 需要会执行的过程奖励模型

**论文：** [Rewarding the Scientific Process: Process-Level Reward Modeling for Agentic Data Analysis](https://arxiv.org/abs/2604.24198)  
**作者：** Zhisong Qiu, Shuofei Qiao, Kewei Xu, Yuqi Zhu, Lun Du, Ningyu Zhang, Huajun Chen  
**机构：** 浙江大学、蚂蚁集团  
**日期：** arXiv, 2026-04-27  
**全文：** [arXiv HTML](https://arxiv.org/html/2604.24198v1)

**核心图表：**

![DataPRM framework](https://arxiv.org/html/2604.24198v1/x5.png)

*图 3：DataPRM 的轨迹生成、step-level annotation、多轮环境交互和反思式三值奖励。*

![DataPRM RL results](https://arxiv.org/html/2604.24198v1/x7.png)

*图 5a：使用过程监督奖励做 agentic RL 后，在 DABench 和 TableBench 上的结果。*

![DataPRM entropy dynamics](https://arxiv.org/html/2604.24198v1/x9.png)

*图 5c：过程奖励与 outcome-only 奖励下的 entropy 动态。*

**图表线索：** 表 1 展示 grounding error 与 silent error 的代表案例；表 2 比较 DataPRM 与通用 PRM、LLM-as-judge、自奖励基线；表 3 消融环境交互、多轮交互和 reflection-aware reward。

**我读到的重点：** DataPRM 的切入点很实际：数据分析 agent 的错误不等于普通数学推理错误。论文先指出两个失败模式。第一，grounding errors 可能只是探索过程中猜错列名或 schema，最终可以通过执行反馈修正，通用 PRM 却容易把它当成致命错误。第二，silent errors 更危险：代码能跑通，但逻辑错了，静态读代码的 PRM 很难发现。

因此 DataPRM 被设计成 environment-aware generative PRM：它自己也用 ReAct 风格，多轮调用代码环境验证当前 step，再给出 step 级分数和理由。奖励策略上，它区分可修复的 grounding error 与不可恢复错误，避免过早惩罚必要探索。

实验里，DataPRM 虽然是 4B 规模，却在 Best-of-N 设置下超过更大的 GenPRM-32B、Qwen2.5-Math-PRM-72B、DeepSeek-V3.2 judge 和 Qwen3-235B self-rewarding。扩展到 Beam Search 和 DVTS 后，它也更抗 reward hacking。用于 agentic RL 时，过程监督模型在 DABench 达到 78.73%、TableBench 达到 64.84%，超过 SFT 和 outcome-only reward；entropy 曲线也显示 outcome-only 更容易塌缩。

**我会优先看这篇的原因：** 它直接对应 data agent 的核心难点：不是让模型“看起来会分析”，而是让验证器能进入数据环境，检查中间步骤是否真的对。

**局限和问题：** 执行式 verifier 带来额外成本和安全边界问题；多轮环境交互本身也可能被错误工具状态误导。论文的结论强依赖 DABStep、ScienceAgentBench、DABench、TableBench 这些任务分布。下一步我会看 DataPRM 能否迁移到更杂乱的企业表格、Notebook 历史状态和真实文件系统。

**主题连接：** data agents、过程奖励模型、可执行验证、agentic RL。

## 5. EnterpriseDocBench：企业文档 AI 不能只评一个环节

**论文：** [Benchmarking Complex Multimodal Document Processing Pipelines: A Unified Evaluation Framework for Enterprise AI](https://arxiv.org/abs/2604.26382)  
**作者：** Saurabh K. Singh, Sachin Raj  
**机构：** Oracle、独立研究者  
**日期：** arXiv, 2026-04-29  
**全文：** [arXiv HTML](https://arxiv.org/html/2604.26382v1)

**核心图表线索：**

- 表 1：EnterpriseDocBench 的公开数据来源与领域分布。
- 表 2：BM25、dense embedding、hybrid fusion 三类 pipeline 在解析、索引、检索、生成四个维度上的结果。
- 表 3：不同上下文长度区间的 hallucination rate。

**我读到的重点：** 这篇不主打新模型，而是把企业文档 AI 作为多阶段 pipeline 来评估：parse、index、retrieve、generate。数据来自 1,459 份许可允许研究使用和再分发的公开文档，覆盖多个企业领域；实验比较 BM25、dense embedding、hybrid retrieval，并固定生成器。

最有意思的结果是“级联质量”假设没有得到强支持。论文报告 parsing 到 retrieval 的 Pearson r 为 0.14，parsing 到 generation 为 0.17，retrieval 到 generation 接近 0.02。作者很谨慎地说这不证明因果独立，因为 parser 固定、generator 共享、指标是自动 proxy；但它至少提醒我们，文档 RAG 的失败不总是线性地从上一环传到下一环。

第二个结果是 hallucination 与上下文长度不是简单单调关系。中等长度文档 hallucination rate 最低，短文档和很长文档都更高；very long 样本量较小，所以作者只把它称为方向性证据。第三个结果更像产品提醒：factual accuracy on stated claims 较高，但 answer completeness 平均只有 0.40，说明系统“说出的部分常常对”，但漏掉很多内容。

**我会优先看这篇的原因：** 文档智能很容易被单点 benchmark 误导。实际系统里，解析、检索、生成、引用和完整性都可能独立出问题。EnterpriseDocBench 虽然还粗糙，但它把评估对象从“模型答题”拉回“系统流水线”。

**局限和问题：** 领域不平衡明显，General 占比接近一半，Legal 和 Manufacturing 样本少；目前只覆盖英文；自动 hallucination 检测的一致性还有限；ColPali、ColQwen2 和 agentic routing 只是 reference architecture，还没有实测。下一步值得追踪它是否加入人工并行评测、多 parser 对照和更均衡的企业查询。

**主题连接：** document intelligence、RAG evaluation、可审计企业 AI、pipeline-level benchmark。

## 阅读优先级和下一步问题

如果只精读两篇，我会先看 **ClawGym** 和 **DataPRM**。前者给出了 agentic training 的完整工程闭环，后者把 data agent 的过程验证讲得足够具体。World2VLM 是 world model 方向本期最值得追的技术路线，因为它把世界模型从推理工具变成训练教师。FutureWorld 则值得持续观察，因为真实延迟反馈会迫使 agent 研究面对数据更新、答案揭晓和概率校准。EnterpriseDocBench 我会当作 document intelligence 评测线索继续跟踪。

后续我想重点追三个问题：

1. verifier 会不会成为新的 shortcut：agent 是否学会满足 verifier，而不是完成真实任务？
2. world model teacher 的可靠性如何量化：能否按任务、按样本估计 teacher uncertainty？
3. 文档智能系统的完整性如何评：factual accuracy 高但 answer completeness 低时，用户侧风险该如何表达？

## English Version {#english-version}

# From Closed-Loop Agent Training to Auditable Data Intelligence

> **TL;DR:** This issue focuses on five late-April 2026 papers. The shared thread is not larger chat models, but better closed loops: agents trained in executable workspaces, predictors rewarded by real-world outcomes, VLMs taught by world models, data-analysis agents supervised by executable process reward models, and enterprise document systems evaluated as pipelines. I found no sufficiently relevant papers in the last 8 hours, so I expanded the search to the last 3-7 days and prioritized papers from April 29, 2026.

## 1. ClawGym

**Paper:** [ClawGym: A Scalable Framework for Building Effective Claw Agents](https://arxiv.org/abs/2604.26904)  
**Authors:** Fei Bai, Huatong Song, Shuang Sun, Daixuan Cheng, Yike Yang, Chuan Hao, Renyuan Li, Feng Chang, Yuan Wei, Ran Tao, Bryan Dai, Jian Yang, Wayne Xin Zhao  
**Institutions:** Renmin University of China, IQuest Research, Beihang University  
**Date:** arXiv, April 29, 2026  
**Full text:** [arXiv HTML](https://arxiv.org/html/2604.26904v1)

**Key figures:** Figure 1 shows the ClawGym-SynData synthesis and filtering pipeline; Figure 3 shows RL curves under sandbox-parallel rollouts; Table 6 is the main benchmark comparison.

ClawGym targets personal workspace agents that operate over local files, tools, and persistent state. The paper links three pieces: 13.5K filtered synthetic workspace tasks, ClawGym-Agents trained from black-box rollout trajectories and GRPO, and a 200-instance ClawGym-Bench with code-based and rubric-based verification. I care about this one because it treats agent training as executable workflow training, not just dialogue imitation.

The main caveat is reproducibility: the resources are described as forthcoming. The benchmark will also depend heavily on verifier design and how well synthetic workspaces approximate real user messiness.

## 2. FutureWorld

**Paper:** [FutureWorld: A Live Environment for Training Predictive Agents with Real-World Outcome Rewards](https://arxiv.org/abs/2604.26733)  
**Authors:** Zhixin Han, Yanzhi Zhang, Chuyang Wei, Maohang Gao, Xiawei Yue, Kefei Chen, Yu Zhuang, Haoxiang Guan, Jiyan He, Jian Li, Yitong Duan, Yu Shi, Mengting Hu, Shuxin Zheng  
**Institutions:** Nankai University, Chinese Academy of Sciences, Zhongguancun Academy, Tsinghua University  
**Date:** arXiv, April 29, 2026  
**Full text:** [arXiv HTML](https://arxiv.org/html/2604.26733v1); [dataset](https://huggingface.co/datasets/PredictingFuture/FutureWorld)

**Key figures:** Figure 2 shows question construction; Figure 3 shows the delayed reward loop; Figure 4 tracks checkpoint performance over days.

FutureWorld frames live future prediction as an agent environment. Each day it releases prediction questions, logs search-and-reasoning trajectories, retrieves outcomes later, and trains with negative Brier loss through GRPO. The important move is making answer leakage harder by using real events whose outcomes are not known when the model acts.

The paper is honest about operational friction: many questions are unresolved at the scheduled retrieval time, and discarding them wastes signal. I will watch whether this line develops longer-delay replay and better handling of source-dependent calendar effects.

## 3. World2VLM

**Paper:** [World2VLM: Distilling World Model Imagination into VLMs for Dynamic Spatial Reasoning](https://arxiv.org/abs/2604.26934)  
**Authors:** Wanyue Zhang, Wenxiang Wu, Wang Xu, Jiaxin Luo, Helu Zhi, Yibin Huang, Shuo Ren, Zitao Liu, Jiajun Zhang  
**Institutions:** Institute of Automation, CAS; UCAS; Tsinghua University; Harbin Institute of Technology; Wuhan AI Research  
**Date:** arXiv, April 29, 2026  
**Full text:** [arXiv HTML](https://arxiv.org/html/2604.26934v1); [code](https://github.com/WanyueZhang-ai/World2VLM); [dataset](https://huggingface.co/datasets/WanyueZhang/World2VLM)

**Key figures:** Figure 2 gives the framework; Figure 3 shows scaling with world-model-generated data; Tables 2-4 contain the main results and ablations.

World2VLM uses a world model as a training-time teacher. It generates action-aligned future views, then trains a VLM on both inverse tasks, recovering the action behind a view change, and forward tasks, predicting action consequences. The result is standard VLM inference at deployment, rather than expensive test-time imagination.

The most useful limitation is teacher reliability. SVC and HY-WorldPlay inject different biases: one is more camera-consistent, the other more perceptually natural. Future work should weight or filter teacher samples by reliability rather than treating all retained samples equally.

## 4. DataPRM

**Paper:** [Rewarding the Scientific Process: Process-Level Reward Modeling for Agentic Data Analysis](https://arxiv.org/abs/2604.24198)  
**Authors:** Zhisong Qiu, Shuofei Qiao, Kewei Xu, Yuqi Zhu, Lun Du, Ningyu Zhang, Huajun Chen  
**Institutions:** Zhejiang University, Ant Group  
**Date:** arXiv, April 27, 2026  
**Full text:** [arXiv HTML](https://arxiv.org/html/2604.24198v1)

**Key figures:** Figure 3 shows the DataPRM framework; Figure 5a shows RL benchmark results; Figure 5c shows entropy dynamics under process rewards.

DataPRM argues that data-analysis agents need process reward models that can execute and inspect intermediate states. General PRMs often punish recoverable grounding errors, such as wrong schema guesses, while missing silent errors where code runs but logic is wrong. DataPRM uses a ReAct-style, environment-aware verifier with multi-turn code interaction and reflection-aware rewards.

The paper reports that process-supervised RL outperforms SFT and outcome-only reward on DABench and TableBench, while avoiding the entropy collapse seen with outcome-only training. The remaining risk is cost and transfer: executable verification is powerful, but real enterprise notebooks and file systems are messier than curated benchmarks.

## 5. EnterpriseDocBench

**Paper:** [Benchmarking Complex Multimodal Document Processing Pipelines: A Unified Evaluation Framework for Enterprise AI](https://arxiv.org/abs/2604.26382)  
**Authors:** Saurabh K. Singh, Sachin Raj  
**Institutions:** Oracle, Independent  
**Date:** arXiv, April 29, 2026  
**Full text:** [arXiv HTML](https://arxiv.org/html/2604.26382v1)

**Key figure pointers:** Table 1 lists the public document sources; Table 2 compares BM25, dense retrieval, and hybrid retrieval; Table 3 breaks hallucination rate by context length.

EnterpriseDocBench evaluates document AI as a pipeline: parsing, indexing, retrieval, and generation. The most interesting result is that inter-stage correlations are weak in this setup, with retrieval-to-generation near zero. The paper is careful not to overclaim causality, because parsing and generation are fixed, but the result pushes against a simple linear cascade story.

I also care about the completeness finding. Factual accuracy on stated claims can look good while answer completeness remains low. That is exactly the kind of deployment risk that single-answer benchmarks often hide.

## What I Will Track Next

The next questions are practical. Can verifiers avoid becoming shortcuts? Can world-model teacher reliability be estimated per sample and per task? Can document intelligence benchmarks report completeness and auditability as first-class failure modes rather than secondary notes?
